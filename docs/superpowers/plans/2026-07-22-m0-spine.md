**Goal:** A running `composerd` daemon with event-sourced sessions and one OpenAI-compatible
provider adapter, plus a minimal VSCodium chat panel â€” chat with a pinned model from the
editor, kill the window, reattach, nothing lost.

**Architecture:** Rust cargo workspace (`daemon/`): `ledger` crate (append-only JSONL event
store), `gateway` crate (OpenAI-compatible SSE chat adapter), `composerd` binary (config,
bearer-token auth, axum HTTP+SSE API, orchestrator turn loop, CLI ground-truth readback).
TypeScript extension (`extension/`): thin client reading the daemon's state dir for
discovery, one webview chat view. Oracle: hermetic assert scripts under `tests/oracle/`
driven by forgeloop specs (no live LLM in the gate â€” a stdlib stub server plays the model).

**Tech Stack:** Rust 1.96 (tokio, axum 0.8, serde, reqwest/rustls, ulid), Node 22 +
TypeScript 5 + esbuild, Python 3 stdlib (oracle stub), bash asserts, forgeloop engine
(`python3 -m forgeloop run`).

## Global Constraints

- Event schema string is exactly `forgeloop.composer.event.v1` (spec Â§4).
- State dir: `$FORGE_COMPOSER_STATE_DIR` if set, else `~/.local/share/forge-composer`.
  Every test and oracle MUST set `FORGE_COMPOSER_STATE_DIR` to a temp dir (hermeticity).
- API binds `127.0.0.1` only. Every route except `GET /health` requires
  `Authorization: Bearer <token>` where token is the contents of `<state>/auth.token`
  (created 0600 on first start). Missing/wrong token â†’ HTTP 401.
- Ledger files are append-only; `append` fsyncs before returning. No event is ever
  rewritten or deleted.
- Secrets loaded from provider config never appear in any ledger byte: the Redactor scrubs
  every serialized event body before write, replacing each secret with `[REDACTED]`.
- No dependency additions beyond `daemon/Cargo.toml [workspace.dependencies]` without
  Architect sign-off. No `unwrap()`/`expect()` on data that crosses a process or network
  boundary (Â§9 least-privilege/errors-handled pack will scan for it).
- Commit after every green task with a `feat:`/`test:`/`chore:` conventional message.

```rust
pub const SCHEMA: &str = "forgeloop.composer.event.v1";

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize, PartialEq)]
pub struct Event {
    pub v: String,
    pub seq: u64,          // 1-based, per session, dense
    pub ts: String,        // RFC3339 UTC, e.g. 2026-07-22T22:31:04Z
    pub session: String,   // ULID string
    pub actor: String,     // "human" | "orchestrator" | "system"
    pub kind: String,      // "message" | "usage" | "error"
    pub provenance: String,// "trusted" | "untrusted"
    pub body: serde_json::Value,
}

#[derive(Debug, Clone, Default)]
pub struct Redactor { secrets: Vec<String> }
impl Redactor {
    pub fn new(secrets: Vec<String>) -> Self;   // drops empty strings
    pub fn scrub(&self, text: &str) -> String;  // each secret -> "[REDACTED]"
}

pub struct SessionStore { /* root: PathBuf, redactor: Redactor */ }
impl SessionStore {
    pub fn new(root: std::path::PathBuf, redactor: Redactor) -> Self; // root = <state>/sessions
    pub fn create_session(&self) -> anyhow::Result<String>;
    pub fn list_sessions(&self) -> anyhow::Result<Vec<String>>;      // sorted
    pub fn append(&self, session: &str, actor: &str, kind: &str,
                  provenance: &str, body: serde_json::Value) -> anyhow::Result<Event>;
    pub fn read(&self, session: &str, since_seq: u64) -> anyhow::Result<Vec<Event>>; // seq > since_seq
}
```

Storage: `<root>/<session-ulid>/ledger.jsonl`, one JSON event per line. `append` computes
`seq` = last seq + 1 (scan or cached), serializes, scrubs via Redactor, writes line,
`sync_all()`. Concurrency: wrap per-store `std::sync::Mutex` (M0 is single-process).
