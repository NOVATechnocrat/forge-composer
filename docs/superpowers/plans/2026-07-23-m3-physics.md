# M3 "Physics" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire forgeloop physics into the Composer: a Judge bridge (`run_gate` tool) whose `verdict` events are file-pointer copies of run journals (never model-synthesized, design §4 / Law 4), a bounded provider escalation chain on stall (design §6), and verdict rendering in the extension.

**Architecture:** A new `forgeloop_bridge` module in `composerd` shells out to `harness/journal-gate.sh` in a configured forgeloop checkout (read-only toward that tree, D8 — the mint hard-deny already exists in the policy crate), parses the printed `evidence: runs/<ts>.jsonl` pointer, reads the journal's last `decision` line, and appends a `verdict` event with actor `judge`. The gateway call in the agent loop gains a chain: on transport/HTTP failure it falls through role-named fallback tiers, recording each hop as an `error` event with `escalated_to`. The extension renders `verdict` events as pass/fail badge cards.

**Tech Stack:** Rust (tokio process, serde), bash fixture scripts in the oracle, TypeScript webview rendering.

**Sealed-oracle contract (read first):** `tests/oracle/assert-m3-physics.sh` + `tests/oracle/stub-llm-m3.py` are authored, red-validated, and human-minted BEFORE this plan executes. **Never edit either file.** Build until `bash tests/oracle/assert-m3-physics.sh` prints `M3-PHYSICS-OK`. Event kinds, field names, and tool schemas below are what the oracle asserts.

## Global Constraints

- `composerd` never writes into the forgeloop tree and has no mint code path (D8). The bridge only *reads* the journal and *runs* `harness/journal-gate.sh` as a subprocess with a scrubbed environment.
- Verdict events are NEVER synthesized: if the gate produces no journal evidence, the daemon appends an `error` event and NO `verdict` event.
- `verdict` events: `actor:"judge"`, `provenance:"trusted"`, body `{"oracle_id": <gate target>, "decision": <from journal>, "journal_path": <absolute path>, "intent": <from journal>}`.
- Escalation is bounded: one pass through the configured chain, never a retry loop.
- `run_gate` is orchestrator-only (chain of command, §8.5) — `Deny` for subagent sessions, schema not offered to them.
- Do not change sealed surfaces: `GET /sessions` shape, existing event bodies, existing tool schemas.
- No new crate dependencies.
- Commit after every task with the exact message given. Another agent may work in `extension/` concurrently when the daemon agent runs — `git add` only your own files; retry once on index.lock contention.

/// The role's provider config followed by its escalation tiers, in order.
pub fn resolve_chain(
    cfg: &Config,
    role: &str,
) -> anyhow::Result<Vec<(String, gateway::ProviderConfig)>> {
    let mut out = vec![(role.to_string(), resolve_role(cfg, role)?)];

        out.push((next.clone(), resolve_role(cfg, next)?));
    }
    Ok(out)
}

```rust
pub struct GateOutcome {
    /// Some when the gate printed journal evidence AND the journal parsed.
    pub verdict: Option<GateVerdict>,
    pub exit_code: i32,
    /// Capped combined stdout+stderr for the tool result / error event.
    pub output: String,
}

pub struct GateVerdict {
    pub decision: String,      // journal's last "decision" value
    pub intent: String,        // journal's last "intent" value ("" if absent)
    pub journal_path: std::path::PathBuf, // absolute
}

/// Run `bash <dir>/harness/journal-gate.sh <target>` with a scrubbed env,
/// parse the LAST `evidence: <path>.jsonl` occurrence from stdout, read that
/// journal (relative paths resolved against <dir>), and extract the last
/// JSONL line carrying a "decision" key. NEVER fabricates a verdict: any
/// parse/read miss yields verdict=None.
pub async fn run_gate(
    dir: &std::path::Path,
    target: &str,
    scrub_names: &[String],
    timeout_secs: u64,
    cap: usize,
) -> anyhow::Result<GateOutcome>
```

- Target validation: `target` must match `^[a-z0-9][a-z0-9-]*$` — else `Err` (it is spliced into a command line).
- Env scrub: spawn with a filtered environment — drop any var whose NAME contains `KEY`, `TOKEN`, `SECRET`, `PASSWORD`, `CREDENTIAL` (case-insensitive) or appears in `scrub_names` (mirror `tools::terminal`'s scrub — read `daemon/crates/tools/src/terminal.rs` first; if its helper is private, replicate the filter here, do NOT change the tools crate's public surface).
- Evidence regex (on stdout only): the last match of `evidence: (\S+\.jsonl)`. Resolve relative against `dir`.
- Journal parse: for each line, `serde_json::from_str::<serde_json::Value>`; keep the LAST line with a `"decision"` string key. Missing file / no decision line → `verdict: None`.
- Timeout: kill the child and return `Ok(GateOutcome { verdict: None, exit_code: -1, output: <partial + " [timeout]"> })`.

```rust
//! forgeloop bridge — runs journal gates by subprocess and copies verdicts
//! from journals on disk. Read-only toward the forgeloop tree (design D8);
//! there is deliberately NO code path here that can invoke mint.sh.

pub async fn run_gate(
    dir: &std::path::Path,
    target: &str,
    scrub_names: &[String],
    timeout_secs: u64,
    cap: usize,
) -> anyhow::Result<GateOutcome> {
    if target.is_empty()
        || !target
            .chars()
            .all(|c| c.is_ascii_lowercase() || c.is_ascii_digit() || c == '-')
        || target.starts_with('-')
    {
        anyhow::bail!("invalid gate target: {target:?}");
    }
    let script = dir.join("harness/journal-gate.sh");
    if !script.exists() {
        anyhow::bail!("no journal-gate.sh under {}", dir.display());
    }
    let mut cmd = tokio::process::Command::new("bash");
    cmd.arg(&script)
        .arg(target)
        .current_dir(dir)
        .stdout(std::process::Stdio::piped())
        .stderr(std::process::Stdio::piped())
        .env_clear();
    for (k, v) in std::env::vars() {
        let upper = k.to_uppercase();
        let secret = ["KEY", "TOKEN", "SECRET", "PASSWORD", "CREDENTIAL"]
            .iter()
            .any(|s| upper.contains(s))
            || scrub_names.iter().any(|n| n == &k);
        if !secret {
            cmd.env(k, v);
        }
    }
    // spawn, wait with tokio::time::timeout, kill on expiry, collect output,
    // cap combined stdout+stderr to `cap` bytes, then:
    //   evidence = last regex-free scan of stdout lines for "evidence: " ...
    //   (find "evidence: ", take following whitespace-terminated token,
    //    require ".jsonl" suffix)
    //   journal_path = dir.join(token) unless token is absolute; canonicalize
    //   parse last decision line -> GateVerdict
    // Return GateOutcome.
}
```

(Do the evidence scan with plain string ops — no regex dependency.)

## Self-Review Notes (already applied)

- **Law 4 as code:** the bridge type makes fabrication impossible to express — `GateVerdict` only exists when a journal line was read from disk; the no-evidence path is `Option::None` and an `error` event. The oracle's red-gate anchor asserts a session with a failed gate has an `error` event and ZERO `verdict` events.
- **Mint stays sealed:** no new code path touches `mint.sh`; the existing policy hard-deny (`forgeloop/harness/mint.sh` substring, non-overridable) is re-proven end-to-end by the oracle's mint-canary anchor (the fixture's `mint.sh` writes a `MINTED` file if it ever runs — it must not exist afterward).
- **Escalation stays honest:** each hop is a ledgered `error` event with `escalated_to` — no silent rerouting (the Cursor failure D6 names). Cost metering follows the tier that actually answered.
- **Injection surface:** gate targets are `[a-z0-9-]`-validated before touching a command line; gate output enters prompts only via the existing untrusted tool-result framing.
- **YAGNI held:** no gate-progress streaming (the tool result is enough for v1), no journal browser in the extension, no per-tier retry counts (one pass through the chain).
